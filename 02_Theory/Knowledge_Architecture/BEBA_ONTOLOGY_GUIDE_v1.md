# BEBA Ontology Guide v1

Status: canonical working architecture

## Purpose

This guide defines how concepts are represented inside BEBA.

Ontology is not a literature review. Ontology represents the current scientific consensus adopted by the BEBA research programme.

## Concept File Template

Each concept should contain:

### Canonical Name

Stable name used across the project.

### Definition

Preferred working definition. Alternative definitions may be documented separately.

### Scientific Domain

Examples:
- Information Economics
- Behavioural Science
- Marketing
- Nonprofit Management
- Psychometrics

### BEBA Layer

Allowed layers:
- Epistemic Layer
- Information Economics Layer
- Institutional Layer
- Signal Layer
- Brand Equity Layer
- Behavioural Layer
- Enactment Layer
- Resource Layer
- Boundary Condition

### Parent Concepts

More general concepts.

### Child Concepts

More specific concepts.

### Related Concepts

Non-hierarchical relations.

### Core Claims

Claims directly associated with the concept.

### Evidence Base

Evidence supporting or questioning the core claims.

### Boundary Conditions

Contexts where the concept weakens, changes, or does not apply.

### Competing Explanations

Alternative theoretical explanations.

### Operationalisations

Typical constructs and variables used to measure the concept.

### Existing Scales

Known scales or item families.

### Dissertation Use

Chapters, papers, and figures where the concept is used.

### Confidence

High, Medium, or Low. Confidence refers to the ontology entry, not the entire theory field.

## Ontology Principles

1. Ontology changes slowly.
2. Claims change more frequently.
3. Evidence changes most frequently.
4. Candidate ontology entries are never automatically canonical.
5. AI-generated ontology candidates require manual review.
6. Every canonical concept must be traceable to evidence, claims, or theory.

## Status Levels

- candidate: proposed by literature extraction or AI synthesis.
- active: used in current working theory.
- canonical: stable for the dissertation version.
- superseded: replaced but archived.

## Naming Rule

Concept files should use lowercase kebab-case where possible, for example:

- epistemic-uncertainty.md
- brand-equity.md
- warm-glow.md
- donation-intention.md

## Separation Rule

A concept file is not a chapter and not a review. It should remain compact, structured, and reusable.
