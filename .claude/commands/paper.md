# /paper

## Purpose

Develop or review an article spin-off from the BEBA monograph.

## Usage

`/paper 06_Papers/<paper-id>/`

## Instructions

1. Read the target paper folder.
2. Read `12_PublicationStrategy/roadmap.md`.
3. Identify which BEBA knowledge objects support the paper.
4. Check consistency with monograph terminology and ConstructLedger.
5. Suggest structure, argument and contribution.

## Output

A paper development note or revised section.

## Required Sections

1. Target journal / audience if known
2. Core contribution
3. Relationship to monograph
4. Required evidence
5. Theory objects used
6. Empirical outputs required
7. Risks
8. Next writing steps

## Rule

Papers may condense or specialise monograph logic, but should not introduce inconsistent construct definitions without ADR or DecisionLog entry.
