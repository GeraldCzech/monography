# /integrate

## Purpose

Assess how new claims, evidence or concept files affect the BEBA theory architecture.

## Usage

`/integrate <file-or-directory>`

## Instructions

1. Read `CLAUDE.md`.
2. Read `.claude/agents/theory-architect.md`.
3. Read `02_Theory/Knowledge_Architecture/BEBA_INFORMATION_MODEL_v1.md`.
4. Read `02_Theory/Knowledge_Architecture/BEBA_RELATIONSHIP_MODEL_v1.md`.
5. Analyse whether the input:
   - supports existing BEBA claims,
   - contradicts existing claims,
   - adds boundary conditions,
   - introduces a bypass route,
   - requires new concepts,
   - changes model competition.

## Output

Create a Theory Integration Unit in:

`02_Theory/Knowledge/Theory_Integration/<topic>.md`

## Required Sections

1. Input reviewed
2. Core theoretical implication
3. BEBA layer affected
4. Claims supported
5. Claims challenged
6. New candidate claims
7. Boundary conditions
8. Model competition implications
9. Ontology candidate implications
10. Recommendation

## Rule

Do not change canonical ontology. Create candidates and recommendations only.
