# /ontology

## Purpose

Review ontology candidates and concept files against the BEBA ontology guide.

## Usage

`/ontology <candidate-or-concept-file>`

## Instructions

1. Read `02_Theory/Knowledge_Architecture/BEBA_ONTOLOGY_GUIDE_v1.md`.
2. Read `02_Theory/Knowledge_Architecture/BEBA_RELATIONSHIP_MODEL_v1.md`.
3. Review the candidate or concept.
4. Assess whether it should remain candidate, become active, become canonical, or be superseded.

## Output

Create an ontology review note in:

`02_Theory/Knowledge/Ontology_Candidates/review-<topic>.md`

## Required Sections

1. Concept reviewed
2. Definition quality
3. BEBA layer fit
4. Evidence base
5. Relationship clarity
6. Boundary conditions
7. ConstructLedger implications
8. Glossary implications
9. Recommendation

## Rule

Do not promote candidates automatically. Provide recommendation only.
