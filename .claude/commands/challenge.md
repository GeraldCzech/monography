# /challenge

## Purpose

Critically challenge a theory, chapter, claim or model.

## Usage

`/challenge <file>`

## Instructions

1. Read `.claude/agents/reviewer-simulator.md`.
2. Read the target file.
3. Challenge the core assumptions.
4. Identify competing explanations.
5. Identify missing boundary conditions.
6. Identify unsupported claims.
7. Identify possible reviewer objections.

## Output

Create a critique file next to the source or in:

`09_ResearchCompass/challenges/`

## Required Sections

1. Core argument reconstructed
2. Strongest version of the argument
3. Weakest assumptions
4. Competing explanations
5. Missing evidence
6. Boundary conditions
7. Reviewer objections
8. Recommended revisions

## Rule

Do not rewrite the source unless explicitly instructed.
